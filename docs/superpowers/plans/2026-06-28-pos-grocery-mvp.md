# POS Grocery MVP Slice Plan

Date: 2026-06-28

## Goal

Build the remaining POS Grocery MVP in verifiable vertical slices:

- Full store settings UI
- Full user CRUD UI/API
- Product CRUD UI/API
- Product image upload metadata through Cloudinary integration point
- Inventory receiving and stock count adjustment
- Inventory Excel export with formatted worksheet
- POS barcode checkout
- Stock deduction on sale completion
- Receipt data and print-friendly UI
- Sales reports
- Dashboard best sellers and best sale time ranges
- Report Excel export with formatted worksheet
- Production deploy assessment and real blockers

## Constraints

- Keep working inside `projects/pos-grocery`.
- Do not expose secrets.
- Use npm because both frontend and backend have `package-lock.json`.
- Keep behavior covered by unit/API/component tests.
- Backend currently uses an in-memory repository for foundation tests. MVP behavior may continue there for verified app behavior, while Prisma/Turso persistence remains the next production hardening step.
- Cloudflare Workers production deploy needs a Worker-compatible entry point or adapter. Do not claim successful production deploy unless it is actually deployed and verified.

## Tasks

1. Add backend failing tests for the main business flows:
   - user CRUD
   - product CRUD
   - product image upload metadata
   - inventory receive/count/export
   - checkout/stock deduction/receipt
   - reports/dashboard/export
2. Implement repository types and services to pass those tests.
3. Register MVC-style Express routes in `createApp`.
4. Build frontend pages as a single responsive operations workspace:
   - store settings
   - users
   - products
   - inventory
   - POS checkout
   - receipt
   - reports
   - dashboard
5. Add frontend tests for visible workflows.
6. Run verification:
   - backend lint/typecheck/test/build
   - frontend lint/typecheck/test/build
   - root scripts
7. Assess production deploy:
   - confirm required GitHub/Vercel/Cloudflare secrets
   - confirm Cloudflare Worker compatibility
   - deploy only if credentials and adapter are ready

## Done Criteria

- Tests prove the requested behavior for MVP flows.
- UI exposes every requested module responsively.
- Excel exports are generated by API endpoints with styled worksheets.
- Cloudinary upload path exists and is mocked in tests.
- Final response clearly separates completed local MVP behavior from any production deploy blocker.
